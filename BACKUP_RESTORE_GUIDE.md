# ☁️ Cloud Backup & Restore System

ကျောက်မျက်ရတနာ စီမံခန့်ခွဲမှု App ၏ ဒေတာများကို Cloud ပေါ်တွင် လုံခြုံစွာ သိမ်းဆည်းခြင်းနှင့် ပြန်လည်ရယူခြင်းဆိုင်ရာ လမ်းညွှန်ဖြစ်ပါသည်။

## ၁။ Backup လုပ်ဆောင်ချက်များ
- **အလိုအလျောက် Backup (Daily Auto Backup)**: App ကို အသုံးပြုနေစဉ် နေ့စဉ် တစ်ကြိမ် အလိုအလျောက် Cloud ပေါ်သို့ ဒေတာများ သိမ်းဆည်းပေးမည်။
- **ကိုယ်တိုင် Backup လုပ်ခြင်း (Manual Backup)**: Settings ထဲရှိ "Backup Now" ခလုတ်ကို နှိပ်၍ အချိန်မရွေး ဒေတာများကို သိမ်းဆည်းနိုင်သည်။
- **လုံခြုံမှု (Security)**: Backup ဖိုင်များကို အသုံးပြုသူတစ်ဦးချင်းစီ၏ သီးခြား Firebase Storage Folder များတွင်သာ လုံခြုံစွာ သိမ်းဆည်းထားသည်။

## ၂။ Restore (ဒေတာ ပြန်လည်ရယူခြင်း)
- **Restore from Cloud**: ဖုန်းအသစ်တွင် App ပြန်တင်သည့်အခါ သို့မဟုတ် ဒေတာများ ပျောက်ဆုံးသွားသည့်အခါ နောက်ဆုံးသိမ်းဆည်းထားသော Backup ဖိုင်ကို ပြန်လည်ရယူနိုင်သည်။
- **Overwrite Protection**: Restore မလုပ်မီ လက်ရှိဒေတာများကို အရင်ဆုံး Backup လုပ်ရန် စနစ်မှ သတိပေးမည် ဖြစ်သည်။

## ၃။ Offline Sync Recovery
- **Retry Logic**: Backup လုပ်နေစဉ် အင်တာနက်ပြတ်တောက်သွားပါက အင်တာနက်ပြန်ရသည်နှင့် အလိုအလျောက် ပြန်လည်လုပ်ဆောင်ပေးမည်။
- **Conflict Resolution**: Cloud ပေါ်ရှိ ဒေတာနှင့် Local ရှိ ဒေတာများ ကွဲလွဲမှုရှိပါက အသုံးပြုသူကို ရွေးချယ်ခိုင်းသည့် စနစ်ပါဝင်သည်။

---

## 🛠 Required Firebase Setup
၁။ Firebase Console တွင် **Storage** ကို Enable လုပ်ပါ။
၂။ Security Rules တွင် အသုံးပြုသူတစ်ဦးချင်းစီ၏ Folder ကိုသာ Access ရအောင် သတ်မှတ်ပါ။
၃။ `firebase_storage` package ကို `pubspec.yaml` တွင် ထည့်သွင်းပါ။
