# 🛡️ Enterprise-grade Security Features

ကျောက်မျက်ရတနာ စီမံခန့်ခွဲမှု App ၏ ဒေတာနှင့် လုံခြုံရေးဆိုင်ရာ အင်္ဂါရပ်များ လမ်းညွှန်ဖြစ်ပါသည်။

## ၁။ အသုံးပြုသူ စစ်ဆေးခြင်း (Authentication)
- **Biometric Lock**: လက်ဗွေ (Fingerprint) သို့မဟုတ် မျက်နှာ (FaceID) ကို အသုံးပြု၍ App ကို လုံခြုံစွာ ပိတ်ထားနိုင်သည်။
- **PIN Lock**: Biometric မရရှိနိုင်ပါက ဂဏန်း ၄ လုံး သို့မဟုတ် ၆ လုံးပါသော PIN နံပါတ်ဖြင့် အသုံးပြုနိုင်သည်။
- **Auto-Lock**: App ကို နောက်ကွယ်သို့ ပို့လိုက်သည်နှင့် (သို့မဟုတ်) အချိန်အတန်ကြာ အသုံးမပြုပါက အလိုအလျောက် ပြန်လည်ပိတ်သွားမည်။

## ၂။ ဒေတာ လုံခြုံရေး (Data Security)
- **Encrypted Local Database**: Drift (SQLCipher) ကို အသုံးပြု၍ ဖုန်းထဲတွင် သိမ်းဆည်းထားသော ဒေတာအားလုံးကို Password ဖြင့် ကုဒ်နံပါတ်ပြောင်း (Encrypt) ထားသဖြင့် ဖုန်းပျောက်ဆုံးသွားသော်လည်း ဒေတာများကို ဖတ်၍မရနိုင်ပါ။
- **Secure Token Storage**: API ချိတ်ဆက်ရန် အသုံးပြုသော JWT Token များကို Android Keystore နှင့် iOS Keychain တို့တွင် လုံခြုံစွာ သိမ်းဆည်းထားသည်။

## ၃။ ခွင့်ပြုချက်များ စီမံခြင်း (Role-based Access Control - RBAC)
- **Owner**: စနစ်တစ်ခုလုံး၏ အချက်အလက်များ၊ အမြတ်အစွန်းများနှင့် အသုံးပြုသူများကို စီမံနိုင်သည်။
- **Accountant**: အရောင်းအဝယ်နှင့် ကုန်ကျစရိတ်များကိုသာ ကြည့်ရှု/ပြင်ဆင်နိုင်သည်။
- **Worker**: မိမိနှင့် သက်ဆိုင်သော ကျောက်မျက် ပြုပြင်ခြင်း မှတ်တမ်းများကိုသာ ထည့်သွင်းနိုင်သည်။
- **Broker**: မိမိနှင့် သက်ဆိုင်သော အရောင်းမှတ်တမ်းများကိုသာ ကြည့်ရှုနိုင်သည်။

## ၄။ လုပ်ဆောင်ချက် မှတ်တမ်းများ (Audit Logs)
- ဒေတာတစ်ခုခုကို ပြင်ဆင်ခြင်း၊ ပယ်ဖျက်ခြင်း သို့မဟုတ် အရေးကြီးသော အချက်အလက်များကို ကြည့်ရှုခြင်းတိုင်းတွင် မည်သူ၊ မည်သည့်အချိန်၊ မည်သည့်အရာကို လုပ်ဆောင်ခဲ့သည်ကို စနစ်မှ အလိုအလျောက် မှတ်တမ်းတင်ထားမည်။

---

## 🛠 Required Packages
- `local_auth`: Biometric authentication အတွက်။
- `flutter_secure_storage`: လုံခြုံသော storage အတွက်။
- `sqlcipher_flutter_libs`: Database encryption အတွက်။
